# Implementation Plan: Arrow + Stroke Style (P2-09)

**Branch**: `feat/online-room` | **Date**: 2026-06-27 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/017-arrow-stroke-style/spec.md`

## Summary

Add the Arrow element type (two-point straight arrow with a filled arrowhead) and extend all shapes with a `strokeStyle: 'solid' | 'dashed' | 'dotted'` selector in the style panel. `strokeStyle` is already defined in `@vdt/shared`'s `ElementProps` and rendered via `strokeDashArray` in existing shapes; this feature wires up the Arrow ShapeUtil and adds the UI control to the detail panel and toolbar.

## Technical Context

**Language/Version**: TypeScript 6.x + React 19.x

**Primary Dependencies**: Zustand 5.x, SVG primitives (`stroke-dasharray`, `path` / `polygon` for arrowhead), `canvas/shapes` ShapeUtil registry

**Storage**: `strokeStyle` persisted via existing localStorage sync (no schema changes to `@vdt/shared`)

**Testing**: Vitest 4.x

**Target Platform**: Browser (desktop)

**Project Type**: Web application ‚Äî SVG whiteboard

**Performance Goals**: Arrow rendering instantaneous; style change applies in one frame

**Constraints**: No arrow-to-shape binding (deferred to P2.5); arrowhead is computed, not via SVG `<marker>` element; minimum drag < 2 px discards the shape

**Scale/Scope**: One new ShapeUtil + detail panel control + toolbar button

## Design Details

### 1. Arrow Element Representation

Arrow uses the existing `props.points: [number, number][]` field (already in `ElementProps`):

```
points[0] = [x1, y1]   ‚Üê tail
points[1] = [x2, y2]   ‚Üê head
```

The bounding box (`x`, `y`, `width`, `height` on the `Element`) spans the two points:

```
x      = min(x1, x2)
y      = min(y1, y2)
width  = |x2 - x1|
height = |y2 - y1|
```

This matches how `line` already works in `buildDraftFromPoints`.

### 2. Arrowhead Rendering

Compute a filled triangular arrowhead as an SVG `<polygon>`:

```
direction vector: d = normalize(head - tail)
perpendicular:    p = (-d.y, d.x)
arrowhead tip:    head
arrowhead base L: head - d*ARROW_HEAD_LEN + p*ARROW_HEAD_WIDTH/2
arrowhead base R: head - d*ARROW_HEAD_LEN - p*ARROW_HEAD_WIDTH/2
```

Constants: `ARROW_HEAD_LEN = 12`, `ARROW_HEAD_WIDTH = 8` (in canvas units).

The line is drawn as `<line x1 y1 x2 y2>` (or `<polyline>`). Arrowhead is a separate `<polygon>` filled with `props.strokeColor`.

### 3. Arrow ShapeUtil

```ts
export const arrowShapeUtil: ShapeUtil = {
  type: 'arrow',
  render(el): JSX.Element,  // <g>: <line> + <polygon arrowhead>
  hitTest(el, x, y): boolean,  // distance from point to line segment < threshold (8px)
  getBounds(el): Rect,  // bounding box of [points[0], points[1]]
  resize(el, handle, dx, dy): Partial<Element>,  // update points[0] or points[1] based on handle
}
```

Hit test: point-to-line-segment distance ‚â§ `max(8, strokeWidth / 2)`.

### 4. Arrow Tool in create-shape-tool.ts

Add `'arrow'` to `SHAPE_TOOLS` and handle it in `buildDraftFromPoints` the same way as `'line'`:
- `points: [[start.x, start.y], [current.x, current.y]]`
- `x = min(start.x, current.x)`, `y = min(start.y, current.y)`, etc.

Minimum size check: `sqrt(dx¬≤ + dy¬≤) >= 2` (spec AC-5).

### 5. Stroke Style UI in DetailPanel

Add after the Stroke width control:

```tsx
<label>
  <span>Stroke style</span>
  <select value={props.strokeStyle} onChange={(e) => patch({ strokeStyle: e.target.value as ... })}>
    <option value="solid">Solid</option>
    <option value="dashed">Dashed</option>
    <option value="dotted">Dotted</option>
  </select>
</label>
```

Show for all element types. When `selectedIds.length > 1`, patch all via `updateElements` (leverages P2-08 multi-select work).
