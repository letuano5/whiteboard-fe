# Implementation Plan: Laser Pointer (P1B-04)

**Branch**: `feat/local-editor` | **Date**: 2026-06-26 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/011-laser-pointer/spec.md`

## Summary

Add a transient laser pointer tool that renders a colored trail as the user moves the mouse, stored exclusively in `interaction.laserTrail` (never in elements), with automatic fade-out ~1–1.5s after the last movement. The trail is cleared immediately when switching tools or leaving the canvas.

## Technical Context

**Language/Version**: TypeScript 6.x / React 19.x

**Primary Dependencies**: Zustand 5.x (interaction store already has `laserTrail`), lucide-react (Zap icon for toolbar), SVG (browser-native polyline rendering)

**Storage**: N/A — laser trail is transient, never persisted

**Testing**: Vitest 4.x

**Target Platform**: Web browser (all modern)

**Project Type**: Frontend SPA (React + Vite)

**Performance Goals**: Trail appears within 1 animation frame (~16ms) of pointer move; no performance regression on existing shape rendering

**Constraints**: Trail MUST NOT enter `elements.store`; trail MUST NOT be written to localStorage; fade completes in ≤1.5s after last movement

**Scale/Scope**: 4 files modified, 1 new file created; no new Zustand store; no new ShapeUtil

## Implementation Design

### State additions (minimal)

Add to `InteractionState` in `src/types/interaction.ts`:
```ts
laserFading: boolean;   // true while the trail is fading out (opacity transition)
```

Add to `interaction.store.ts`:
```ts
laserFading: false,
setLaserFading: (v) => set({ laserFading: v }),
```

### laser-tool.ts design

Module-level timer refs (no store); three exported functions:

```ts
// Called from Whiteboard.tsx → handlePointerMove (when tool === 'laser')
onLaserPointerMove(pt: Point): void
  // Push pt to laserTrail (cap at MAX_POINTS=80); reset fade/clear timers:
  //   fadeTimer  → 1000ms → setLaserFading(true)
  //   clearTimer → 1500ms → setLaserTrail([]), setLaserFading(false)

// Called from Whiteboard.tsx → handlePointerLeave (when tool === 'laser')
onLaserPointerLeave(): void
  // Clear timers + immediately call clearLaserTrail()

// Called from Toolbar's chooseTool() when switching away from 'laser'
clearLaserTrail(): void
  // Clear timers + setLaserTrail([]) + setLaserFading(false)
```

### SvgLayer.tsx rendering

Inside the camera `<g>`, read `laserTrail` and `laserFading` from interaction store and render:

```tsx
{laserTrail.length >= 2 && (
  <polyline
    points={laserTrail.map(p => `${p.x},${p.y}`).join(' ')}
    fill="none"
    stroke="#ef4444"
    strokeWidth={3}
    strokeLinecap="round"
    strokeLinejoin="round"
    opacity={laserFading ? 0 : 1}
    style={{ transition: laserFading ? 'opacity 0.5s ease-out' : 'none', pointerEvents: 'none' }}
  />
)}
```

### Whiteboard.tsx changes

- `handlePointerDown`: when `tool === 'laser'`, capture pointer + call `onLaserPointerMove(worldPt)` (start trail)
- `handlePointerMove`: when `tool === 'laser'`, call `onLaserPointerMove(worldPt)`
- `handlePointerUp` / `handlePointerLeave`: when `tool === 'laser'`, call `onLaserPointerLeave()`
- Cursor style: add `tool === 'laser'` → `'crosshair'` to the cursor computation

### Toolbar.tsx changes

- Import `Zap` from `lucide-react`
- Add `{ id: 'laser', label: 'Laser', Icon: Zap }` to TOOLS array
- In `chooseTool()`, add `clearLaserTrail()` call (from `laser-tool.ts`) before switching tool
