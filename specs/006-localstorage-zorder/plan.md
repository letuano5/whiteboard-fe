# Implementation Plan: localStorage Persistence & Z-Order Foundation (P1A-09 + P1A-10)

**Branch**: `feat/local-editor` | **Date**: 2026-06-24 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/006-localstorage-zorder/spec.md`

## Summary

Two P1A features. **P1A-09** adds automatic, debounced localStorage persistence for the committed canvas state (elements + camera) so the scene survives page reloads. Implementation creates `src/sync/local-storage.ts`, which wires into the existing mutation pipeline hook and the camera store subscription, then restores saved state on init. **P1A-10** (z-order foundation) is already implemented in the codebase — the task is test coverage only.

## Technical Context

**Language/Version**: TypeScript 6.x, strict mode, no `any`

**Primary Dependencies**: React 19, Zustand 5.x (`subscribeWithSelector` middleware), Vite 8

**Storage**: Browser native `localStorage` (P1A-09). No additional library.

**Testing**: Vitest 4.x; jsdom environment; `@testing-library/react` for component tests

**Target Platform**: Modern browser (Chrome/Firefox/Safari), single tab (P1A-09 only)

**Project Type**: Web application — collaborative SVG canvas editor

**Performance Goals**: Debounce write ≤ 300 ms from last change; restore before first paint

**Constraints**: localStorage only (no IndexedDB); cross-tab sync deferred to P1B; QuotaExceededError handled silently

**Scale/Scope**: Single room, single user for P1A

## Phase 0: Research

No external research needed. localStorage is a native browser API with no versioning churn. All relevant design decisions are already in CLAUDE.md and the codebase.

**Decision log:**

| Decision | Choice | Rationale |
|---|---|---|
| localStorage key | `VDT_WHITEBOARD_SCENE` | Single constant; easy to clear in DevTools; aligns with project name |
| Save trigger | `registerMutationHook` (elements) + `useCameraStore.subscribe` (camera) | Mutations cover all element changes; camera changes bypass pipeline so need separate subscription |
| Debounce scope | Single shared debounce timer for both triggers | Prevents double-write when user action mutates element and also pans; simplifies tests |
| Restore timing | Before `ReactDOM.createRoot(...).render(...)` in `main.tsx` | Ensures store has correct initial state before any component reads it; avoids a flash of empty canvas |
| Error handling | `try/catch` around `JSON.parse`; empty-canvas fallback | Protects against corrupted data, quota errors, and missing keys without crashing |
| Camera restore | `useCameraStore.getState().setCamera(stored.camera)` | Reuses existing setter; validates zoom clamp via `zoomTo` if needed |
| P1A-10 scope | Tests only (implementation already in codebase) | `SvgLayer` sorts ascending zIndex; `select-tool` sorts descending for hit-test; `createElement` assigns max+1 |

### `src/sync/local-storage.ts` API

```ts
const STORAGE_KEY = 'VDT_WHITEBOARD_SCENE';

interface PersistedScene {
  elements: Element[];
  camera: Camera;
}

// Called once in main.tsx before mount.
// Reads localStorage, restores elements + camera. Returns without error on any failure.
export function initLocalStoragePersistence(): void

// Wires the mutation hook + camera subscription + debounce timer.
// Call once after initLocalStoragePersistence() (also in main.tsx).
// Returns a cleanup function (unregisters hooks; for tests).
export function startLocalStoragePersistence(): () => void
```

### Debounce pattern (no external library)

```ts
let debounceTimer: ReturnType<typeof setTimeout> | null = null;

function scheduleWrite(): void {
  if (debounceTimer !== null) clearTimeout(debounceTimer);
  debounceTimer = setTimeout(flushWrite, DEBOUNCE_MS);
}
```

### Zustand camera subscription

```ts
import { useCameraStore } from '../store/camera.store';

const unsubCamera = useCameraStore.subscribe(
  (state, prevState) => {
    if (state.camera !== prevState.camera) scheduleWrite();
  },
);
```

`camera.store.ts` does NOT use `subscribeWithSelector` middleware, so the two-argument selector overload is unavailable. The vanilla `subscribe(listener)` form receives `(state, prevState)` — manual object-reference comparison `state.camera !== prevState.camera` is correct because every `panBy`/`zoomTo`/`setCamera`/`resetCamera` creates a new `camera` object, making reference inequality a reliable change detector.

### Restore guard (data validation)

```ts
function readScene(): PersistedScene | null {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    if (!raw) return null;
    const parsed: unknown = JSON.parse(raw);
    if (!isValidScene(parsed)) return null;  // type guard
    return parsed;
  } catch {
    return null;
  }
}
```

`isValidScene` is a minimal type guard: checks `parsed` is an object, has `elements` array, has `camera` object with numeric `x/y/zoom`. Does not validate every `Element` field to keep it simple; corrupt individual elements would just render incorrectly rather than crash.
