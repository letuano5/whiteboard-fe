# Implementation Plan: Rotate + Resize for Rotated Shapes (P1B-01)

**Branch**: `feat/local-editor` | **Date**: 2026-06-24 | **Spec**: [spec.md](spec.md)

**Input**: [specs/008-rotate-resize/spec.md](spec.md)

## Summary

Lift the `angle = 0` constraint from Phase 1A. Add a rotate handle to the selection overlay so users can rotate shapes around their bbox center. Fix hit-test and resize to operate in the element's local (un-rotated) coordinate frame so both work correctly when `angle ≠ 0`. Fix line shape rendering to apply the same SVG rotation transform the other shapes already use. All mutations go through `patchElement`; no new files in the mutation pipeline; no store beyond adding `isRotating: boolean` to `InteractionState`.

## Implementation Steps

### Step 1 — Geometry utils (`src/utils/geometry.ts`)

New file. Pure functions:

```ts
import type { Point } from '../types/geometry';

export function rotatePoint(pt: Point, center: Point, angle: number): Point {
  const cos = Math.cos(angle);
  const sin = Math.sin(angle);
  const dx = pt.x - center.x;
  const dy = pt.y - center.y;
  return { x: center.x + dx * cos - dy * sin, y: center.y + dx * sin + dy * cos };
}

export function unrotatePoint(pt: Point, center: Point, angle: number): Point {
  return rotatePoint(pt, center, -angle);
}
```

### Step 2 — `InteractionState` + store

**`src/types/interaction.ts`**: add `isRotating: boolean` to `InteractionState` interface.

**`src/store/interaction.store.ts`**: add `isRotating: false` to `DEFAULT_STATE`; add `setIsRotating: (v: boolean) => void` action.

### Step 3 — Line shape rotation rendering (`src/canvas/shapes/line.tsx`)

Wrap the existing `<polyline>` / `<line>` in a `<g transform={angle !== 0 ? `rotate(${(angle*180/Math.PI)} ${cx} ${cy})` : undefined}>`. Same pattern used by rectangle, ellipse, text, diamond.

### Step 4 — Hit-test un-rotate (`src/canvas/tools/select-tool.ts`)

In `onSelectPointerDown`, before calling `util.hitTest(el, x, y)`, un-rotate the test point around the element center:

```ts
import { unrotatePoint } from '../../utils/geometry';

// inside the loop:
const cx = el.x + el.width / 2;
const cy = el.y + el.height / 2;
const localPt = el.angle !== 0 ? unrotatePoint(worldPt, { x: cx, y: cy }, el.angle) : worldPt;
if (util.hitTest(el, localPt.x, localPt.y)) { ... }
```

No change to ShapeUtil.hitTest methods.

### Step 5 — Resize un-rotate for rotated shapes (`src/canvas/tools/select-tool.ts`)

In `onSelectPointerMove`, when `resizeSession` is active and the element has `angle !== 0`, convert `worldPt` to local space before calling `resizeBoundsFromAnchorAndPointer`:

```ts
import { unrotatePoint } from '../../utils/geometry';

// inside resize branch:
const el = elements.find(e => e.id === draggingId);
if (!el) return;

let effectivePointer = worldPt;
if (el.angle !== 0) {
  const cx = resizeSession.originalBounds.x + resizeSession.originalBounds.width / 2;
  const cy = resizeSession.originalBounds.y + resizeSession.originalBounds.height / 2;
  effectivePointer = unrotatePoint(worldPt, { x: cx, y: cy }, el.angle);
}

const { x, y, width, height, flippedX, flippedY, activeHandle } =
  resizeBoundsFromAnchorAndPointer(resizeSession, effectivePointer);
```

The result `{x, y, width, height}` is in local space, which is what the element stores — correct.

### Step 6 — Rotate interaction functions (`src/canvas/tools/select-tool.ts`)

Add three new exported functions:

**`onRotateHandlePointerDown(worldPt: Point): void`**
- Get selected element from stores.
- Set `draggingId`, `dragStart`, `isRotating = true`.
- Capture initial angle from element (for initial offset calculation).

**Helper**: In `onSelectPointerMove`, add a rotate branch (checked before the resize branch):

```ts
if (isRotating && draggingId && dragStart && el) {
  const cx = el.x + el.width / 2;
  const cy = el.y + el.height / 2;
  // Angle from center to pointer, offset by π/2 so handle starts "above" the shape
  const rawAngle = Math.atan2(worldPt.y - cy, worldPt.x - cx) + Math.PI / 2;
  // Normalize to [-π, π]
  const angle = ((rawAngle + Math.PI) % (2 * Math.PI)) - Math.PI;
  setDraftElement({ ...el, angle });
  return;
}
```

**In `onSelectPointerUp`**: commit rotate:

```ts
if (isRotating && draggingId && draftElement) {
  patchElement(draggingId, { angle: draftElement.angle });
}
setIsRotating(false);
```

### Step 7 — SelectionOverlay with rotate handle (`src/canvas/layers/SvgLayer.tsx`)

1. Apply rotation transform to the entire `<g>` containing the selection ring and resize handles:
   ```tsx
   const angle = element.angle;
   const cx = x + w / 2;
   const cy = y + h / 2;
   const transform = angle !== 0 ? `rotate(${(angle * 180) / Math.PI} ${cx} ${cy})` : undefined;
   return <g transform={transform}>...</g>;
   ```

2. Add a rotate handle inside the same `<g>` at local position `(cx, y - ROTATE_HANDLE_OFFSET)`:
   ```tsx
   const ROTATE_HANDLE_OFFSET = 24;
   // Connector line
   <line x1={cx} y1={y} x2={cx} y2={y - ROTATE_HANDLE_OFFSET} stroke="#3b82f6" strokeWidth={1} />
   // Rotate handle circle
   <circle cx={cx} cy={y - ROTATE_HANDLE_OFFSET} r={5} fill="white" stroke="#3b82f6"
     strokeWidth={1.5} style={{ cursor: 'crosshair' }}
     onPointerDown={e => { e.stopPropagation(); onHandlePointerDown?.('rotate', e); }} />
   ```

3. Widen `onHandlePointerDown` prop type from `ResizeHandleId` to `HandleId`.

### Step 8 — Route rotate handle in Whiteboard (`src/canvas/Whiteboard.tsx`)

Import `HandleId` from interaction types and `onRotateHandlePointerDown` from select-tool. In `handleHandlePointerDown`:

```ts
function handleHandlePointerDown(handle: HandleId, e: React.PointerEvent<SVGCircleElement>) {
  const svgEl = e.currentTarget.closest('svg') as SVGSVGElement | null;
  if (!svgEl) return;
  const rect = svgEl.getBoundingClientRect();
  const worldPt = screenToWorld(e.clientX - rect.left, e.clientY - rect.top, camera);
  if (handle === 'rotate') {
    onRotateHandlePointerDown(worldPt);
    svgEl.setPointerCapture(e.pointerId);
  } else {
    onSelectHandlePointerDown(handle, worldPt);
  }
}
```
