# Implementation Plan: Local Undo / Redo (P1B-06)

**Branch**: `feat/local-editor` | **Date**: 2026-06-25 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/010-undo-redo/spec.md`

## Summary

Add session-local undo/redo (Ctrl/Cmd+Z and Ctrl/Cmd+Shift+Z) to the whiteboard. A new `history.store.ts` holds an undo stack and a redo stack (max 100 entries). History entries are captured via the existing `registerMutationHook` mechanism in `mutation-pipeline.ts`. Each undo/redo applies the inverse snapshot through a new `applySnapshot` pipeline helper, which bumps `version++` to satisfy sync-validity requirements. History is in-memory only (not persisted).

## Technical Context

**Language/Version**: TypeScript 6.x (strict: true)

**Primary Dependencies**: React 19.x, Zustand 5.x (all already in repo; no new packages)

**Storage**: In-memory Zustand store — NOT persisted to localStorage, NOT broadcast to other tabs

**Testing**: Vitest 4.x

**Performance Goals**: Undo/redo completes within one frame (<16 ms); stacks are small arrays

**Constraints**: No new npm packages; max 100 undo steps; history resets on page reload

**Scale/Scope**: Single user, local session; no network involvement in this phase

## Implementation Steps

### Step 1 — Extend MutationEvent with `before` field

In `src/store/mutation-pipeline.ts`, add `before: Element[]` to `MutationEvent`:

```typescript
export interface MutationEvent {
  type: 'create' | 'patch' | 'delete' | 'update';
  elements: Element[];   // after-state (unchanged — existing hooks unaffected)
  before: Element[];     // NEW: element states before the mutation ([] for createElement)
}
```

Update each pipeline function to capture before-state from the store BEFORE writing:
- `createElement`: `before = []`
- `patchElement`: `before = [existing]` (the element as it was before the patch)
- `deleteElements`: `before = [original elements]` (captured before soft-delete)
- `updateElements`: `before = [existing elements]` (captured before the multi-update)

### Step 2 — Add `applySnapshot` to `mutation-pipeline.ts`

New exported function for undo/redo restoration:

```typescript
export function applySnapshot(elements: Element[]): void {
  if (elements.length === 0) return;
  const now = Date.now();
  const bumped = elements.map((el) => ({
    ...el,
    version: el.version + 1,
    versionNonce: nextNonce(),
    updatedAt: now,
  }));
  // Use the store method directly — handles both active and soft-deleted elements
  useElementsStore.getState().updateElements(bumped);
  fireHooks({ type: 'update', before: elements, elements: bumped });
}
```

This is used to restore patch/delete/update operations. Undo-of-create uses the existing `deleteElements`.

### Step 3 — Create `src/store/history.store.ts`

New Zustand store following the repo's curried pattern:

```typescript
import { create } from 'zustand';
import type { Element } from '../types/shared';
import { deleteElements, applySnapshot } from './mutation-pipeline';

export interface HistoryEntry {
  before: Element[];
  after: Element[];
}

const MAX_HISTORY = 100;

// ... create<State & Actions>()(...)
// States: undoStack, redoStack, isApplying
// Actions: push (clears redoStack, enforces maxSize), undo (silent no-op if empty), redo (silent no-op if empty)
```

Key behaviors:
- `push`: appends to undoStack; if length > 100, drops oldest (index 0); clears redoStack (AC-9)
- `undo`: pops undoStack → applies `before` (or `deleteElements` for create undo) → pushes to redoStack; sets `isApplying` true during application
- `redo`: pops redoStack → applies `after` via `applySnapshot` → pushes back to undoStack; sets `isApplying` true during application

### Step 4 — Create `src/sync/history-capture.ts`

```typescript
import { registerMutationHook } from '../store/mutation-pipeline';
import { useHistoryStore } from '../store/history.store';

export function initHistoryCapture(): () => void {
  return registerMutationHook((event) => {
    if (useHistoryStore.getState().isApplying) return;  // suppress re-entry
    useHistoryStore.getState().push({ before: event.before, after: event.elements });
  });
}
```

### Step 6 — Keyboard handler in `Whiteboard.tsx`

New `useEffect` (no dependency on `tool` — undo/redo works in all tool modes):

```typescript
useEffect(() => {
  function handleUndoRedo(e: KeyboardEvent) {
    const target = e.target as HTMLElement;
    if (
      target.tagName === 'INPUT' ||
      target.tagName === 'TEXTAREA' ||
      target.tagName === 'SELECT' ||
      target.isContentEditable
    ) return;    // AC-15: do not intercept text inputs

    const isMod = e.ctrlKey || e.metaKey;
    if (!isMod) return;

    if (e.key === 'z' && !e.shiftKey) {
      e.preventDefault();
      useHistoryStore.getState().undo();
    } else if (e.key === 'z' && e.shiftKey) {
      e.preventDefault();
      useHistoryStore.getState().redo();
    }
  }
  window.addEventListener('keydown', handleUndoRedo);
  return () => window.removeEventListener('keydown', handleUndoRedo);
}, []);
```
