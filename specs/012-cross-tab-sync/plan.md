# Implementation Plan: Cross-Tab Sync (BroadcastChannel)

**Branch**: `feat/local-editor` | **Date**: 2026-06-26 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/012-cross-tab-sync/spec.md`

## Summary

Implement cross-tab synchronization so changes made in one browser tab automatically appear in other tabs of the same origin. The mechanism uses the `BroadcastChannel` Web API to broadcast `Element[]` payloads on every mutation, and a shared `applyRemoteElements` function (with Last-Write-Wins conflict resolution) to apply incoming changes. This `applyRemoteElements` is the **single** implementation that Phase 2 Socket.IO sync will reuse — no duplication.

## Technical Context

**Language/Version**: TypeScript 6.x (strict mode)

**Primary Dependencies**: React 19, Zustand 5, Vite 8 — all already in the project. New dependency: none (`BroadcastChannel` is a built-in Web API, no npm package needed).

**Storage**: localStorage (already wired) — remote changes must also be persisted there.

**Testing**: Vitest 4.x — unit tests for `applyRemoteElements` LWW logic and BroadcastChannel wiring.

**Target Platform**: Modern browser (Chrome 54+, Firefox 38+, Safari 15.4+ — all support `BroadcastChannel`).

**Project Type**: Web application (frontend-only for this feature; no backend changes).

**Performance Goals**: Remote change appears in receiving tab within ~100 ms.

**Constraints**: No new npm packages. No change to the `Element` data model. `applyRemoteElements` must be reusable from Socket.IO (Phase 2) without modification.

**Scale/Scope**: Single whiteboard scene, same-browser tabs only, P1B (local-only, no server).

## Phase 1: Design

### Data Flow

```
Tab A mutation
  → mutation-pipeline fires hook
  → BC hook: if NOT isApplyingRemote() → bc.postMessage({ elements })
  → localStorage hook: saves scene

Tab B BroadcastChannel receives message
  → applyRemoteElements(incoming)
    → LWW filter (skip lower/tied-nonce versions, skip active elements)
    → useElementsStore.setState (merge changes)
    → dispatchMutationEvent({ type: 'update', elements: applied, before: previous })
      → localStorage hook fires → saves to localStorage ✓
      → BC hook: isApplyingRemote() === true → SKIPS (no re-broadcast) ✓
      → history hook: isApplyingRemote() === true → SKIPS (remote changes not undoable) ✓
```

#### `src/sync/apply-remote.ts` — New file

```ts
// Module-level flag prevents re-broadcast and history capture for remote events
let _isApplyingRemote = false;
export function isApplyingRemote(): boolean { return _isApplyingRemote; }

export function applyRemoteElements(incoming: Element[]): void {
  if (incoming.length === 0) return;

  const { elements } = useElementsStore.getState();
  const { draggingId, selectedIds, resizeSession, isRotating, editingId } =
    useInteractionStore.getState();

  // Build set of element IDs currently under active local interaction
  const activeIds = new Set<string>();
  if (draggingId) activeIds.add(draggingId);
  if ((resizeSession !== null || isRotating) && selectedIds.length > 0)
    selectedIds.forEach((id) => activeIds.add(id));
  if (editingId) activeIds.add(editingId);

  const storeMap = new Map(elements.map((e) => [e.id, e]));
  const toApply: Element[] = [];
  const before: Element[] = [];

  for (const el of incoming) {
    if (activeIds.has(el.id)) continue;           // skip active elements (AC-9,10,11)
    const current = storeMap.get(el.id);
    if (!current) {
      toApply.push(el);                           // new element — always add (AC-1)
      before.push(el);
    } else {
      const wins =
        el.version > current.version ||           // higher version wins (AC-5)
        (el.version === current.version && el.versionNonce < current.versionNonce); // tie: lower nonce wins (AC-7)
      if (wins) {
        toApply.push(el);
        before.push(current);
      }
      // else: ignored (AC-6, AC-8)
    }
  }

  if (toApply.length === 0) return;

  // Merge into store: update existing elements, append new ones
  const applyMap = new Map(toApply.map((e) => [e.id, e]));
  const newElements = [
    ...elements.map((e) => applyMap.get(e.id) ?? e),
    ...toApply.filter((e) => !storeMap.has(e.id)),
  ];

  _isApplyingRemote = true;
  try {
    useElementsStore.getState().setElements(newElements);
    dispatchMutationEvent({
      type: 'update',
      elements: toApply,
      before: before.filter((e) => storeMap.has(e.id)), // only "real" befores (not new elements)
    });
  } finally {
    _isApplyingRemote = false;
  }
}
```

#### `src/sync/broadcast-channel.ts` — New file

```ts
const CHANNEL_NAME = 'VDT_WHITEBOARD';

interface BCMessage { elements: Element[]; }

let _channel: BroadcastChannel | null = null;
let _unregisterHook: (() => void) | null = null;

export function initBroadcastChannel(): void {
  if (typeof BroadcastChannel === 'undefined') return; // graceful fallback (AC-13 edge)

  _channel = new BroadcastChannel(CHANNEL_NAME);

  // Receive: apply remote elements (BroadcastChannel does NOT echo to sender — AC-13)
  _channel.onmessage = (event: MessageEvent<BCMessage>) => {
    applyRemoteElements(event.data.elements);
  };

  // Send: broadcast after every local mutation (skip re-broadcast of received remote)
  _unregisterHook = registerMutationHook((e) => {
    if (isApplyingRemote()) return;   // do NOT re-broadcast received remote changes (AC-13)
    if (!_channel) return;
    _channel.postMessage({ elements: e.elements } satisfies BCMessage);
  });
}

export function stopBroadcastChannel(): void {
  _unregisterHook?.();
  _unregisterHook = null;
  _channel?.close();
  _channel = null;
}
```

#### `src/sync/history-capture.ts` — Change

Add `isApplyingRemote()` guard so remote changes are not pushed to the undo stack:

```ts
import { isApplyingRemote } from './apply-remote';

export function initHistoryCapture(): () => void {
  return registerMutationHook((event) => {
    if (useHistoryStore.getState().isApplying) return;
    if (isApplyingRemote()) return;              // remote changes are not locally undoable
    useHistoryStore.getState().push({ before: event.before, after: event.elements });
  });
}
```

### LWW Rule Summary

| Condition | Result |
|-----------|--------|
| `incoming.version > current.version` | Apply incoming (AC-5) |
| `incoming.version < current.version` | Ignore (AC-6) |
| `incoming.version === current.version && incoming.versionNonce < current.versionNonce` | Apply incoming (AC-7) |
| `incoming.version === current.version && incoming.versionNonce >= current.versionNonce` | Ignore (AC-8) |
| `id` not in store | Always apply — new element (AC-1) |
| `id` in `activeIds` | Skip until interaction ends (AC-9, AC-10, AC-11) |

### Active-Element Skip Logic

An element is considered "active" (skipped during `applyRemoteElements`) if:
- Its `id` equals `draggingId` (being dragged), OR
- Its `id` is in `selectedIds` AND (`resizeSession !== null` OR `isRotating === true`), OR
- Its `id` equals `editingId` (text editing in progress)
