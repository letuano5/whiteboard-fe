# Implementation Plan: Zoom + Pan + Infinite Canvas (P1A-06)

**Branch**: `feat/local-editor` | **Date**: 2026-06-24 | **Spec**: [spec.md](spec.md)

## Summary

Add scroll-wheel zoom (pivot-point, clamped [0.1, 8]), Hand-tool pointer pan, middle-mouse-button pan, and Space+drag temporary pan to the Whiteboard SVG layer. All camera mutations go through the existing `camera.store.ts` (`zoomTo`, `panBy`). No new stores, no new npm packages; transient pan drag origin lives in a `useRef` inside `Whiteboard.tsx`.

## Implementation Design

### 1. Scroll-wheel zoom (covers AC-1, AC-2, AC-3, AC-4)

React's synthetic `onWheel` is passive in React 17+ and cannot call `preventDefault` to stop page scroll. Must attach a non-passive native listener via `useEffect`:

```ts
useEffect(() => {
  const el = svgRef.current;
  if (!el) return;
  function handleWheel(e: WheelEvent) {
    e.preventDefault();
    const zoomTo = useCameraStore.getState().zoomTo;
    const camera = useCameraStore.getState().camera;
    const factor = e.deltaY < 0 ? 1.1 : 1 / 1.1;
    const rect = el!.getBoundingClientRect();
    const pivot = { x: e.clientX - rect.left, y: e.clientY - rect.top };
    zoomTo(camera.zoom * factor, pivot);
  }
  el.addEventListener('wheel', handleWheel, { passive: false });
  return () => el.removeEventListener('wheel', handleWheel);
}, []);
```

`zoomTo` already adjusts `(x, y)` so the world point under `pivot` stays fixed (verified in existing implementation).

### 2. Pan mode detection

A pointer event triggers pan mode when ANY of:
- `tool === 'hand'`
- `e.button === 1` (middle mouse button)
- `spaceDown === true`

```ts
function isPanTrigger(e: React.PointerEvent, tool: ToolId, spaceDown: boolean) {
  return tool === 'hand' || e.button === 1 || spaceDown;
}
```

### 3. Pointer pan (covers AC-5, AC-6, AC-7, AC-8, AC-9)

```ts
// Refs (no re-render on pointer-move)
const panStart = useRef<{ x: number; y: number } | null>(null);

// In handlePointerDown — check pan BEFORE existing tool guards
if (isPanTrigger(e, tool, spaceDown)) {
  panStart.current = { x: e.clientX, y: e.clientY };
  setIsPanning(true);             // drives cursor; triggers re-render only on down/up
  e.currentTarget.setPointerCapture(e.pointerId);
  e.preventDefault();
  e.stopPropagation();
  return;
}

// In handlePointerMove — check pan BEFORE existing tool logic
if (panStart.current) {
  const dx = e.clientX - panStart.current.x;
  const dy = e.clientY - panStart.current.y;
  const { camera, panBy } = useCameraStore.getState();
  panBy(-dx / camera.zoom, -dy / camera.zoom);
  panStart.current = { x: e.clientX, y: e.clientY };   // incremental delta
  return;
}

// In handlePointerUp and handlePointerLeave
if (panStart.current) {
  panStart.current = null;
  setIsPanning(false);
  return;
}
```

### 4. Space key temporary pan (covers AC-10, AC-11, AC-12)

```ts
const [spaceDown, setSpaceDown] = useState(false);

useEffect(() => {
  function onKeyDown(e: KeyboardEvent) {
    if (e.code !== 'Space') return;
    const target = e.target as HTMLElement;
    if (
      target.tagName === 'INPUT' ||
      target.tagName === 'TEXTAREA' ||
      target.tagName === 'SELECT' ||
      target.isContentEditable
    ) return;
    e.preventDefault();
    setSpaceDown(true);
  }
  function onKeyUp(e: KeyboardEvent) {
    if (e.code === 'Space') setSpaceDown(false);
  }
  window.addEventListener('keydown', onKeyDown);
  window.addEventListener('keyup', onKeyUp);
  return () => {
    window.removeEventListener('keydown', onKeyDown);
    window.removeEventListener('keyup', onKeyUp);
  };
}, []);
```

### 5. Cursor style (UX polish)

```ts
const [isPanning, setIsPanning] = useState(false);

const cursor =
  isPanning ? 'grabbing'
  : (tool === 'hand' || spaceDown) ? 'grab'
  : undefined;
```

Applied as `<svg style={{ cursor, ... }}>`.

### 6. `svgRef` for non-passive wheel listener

Add `const svgRef = useRef<SVGSVGElement>(null)` and `<svg ref={svgRef} ...>`.

### 7. Existing handler guard ordering

Current `handlePointerDown` starts with `if (!(e.target instanceof SVGElement)) return`. For middle-mouse pan to work on the SVG background (no shape under cursor), this guard must be checked AFTER the pan trigger check, or the guard must allow pan regardless of target.

Decision: Move pan-trigger check to be the first thing in `handlePointerDown` before the SVGElement guard.
