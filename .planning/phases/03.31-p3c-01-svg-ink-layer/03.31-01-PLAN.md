# Plan 03.31-01: Add SVG Ink Shape Rendering

## Objective

Implement `P3C-01` so freehand and highlighter elements render in the existing shared-camera SVG
element layer without introducing a Canvas renderer.

## Target Files

- `packages/shared/src/index.ts`
- `frontend/src/canvas/shapes/ink.tsx`
- `frontend/src/canvas/shapes/index.ts`
- `frontend/src/canvas/shapes/__tests__/registry.test.ts`
- `frontend/src/canvas/shapes/__tests__/shapes.test.tsx`
- `frontend/src/canvas/layers/__tests__/SvgLayer.test.tsx`
- `.planning/phases/03.31-p3c-01-svg-ink-layer/VERIFICATION.md`

## Tasks

1. Add `freehandShapeUtil` and `highlighterShapeUtil` backed by SVG `<path>` rendering.
   - Covers: `AC-1`, `AC-2`
   - Verify: `pnpm --filter whiteboard-fe test -- shapes.test.tsx`

2. Register the ink shape utils in the existing shape registry and update registry tests.
   - Covers: `AC-1`
   - Verify: `pnpm --filter whiteboard-fe test -- registry.test.ts`

3. Add SVG layer acceptance tests proving committed freehand/highlighter elements render under the
   same camera-transformed SVG group with unscaled world-coordinate path data.
   - Covers: `AC-1`, `AC-2`
   - Verify: `pnpm --filter whiteboard-fe test -- SvgLayer.test.tsx`

4. Update stale shared type commentary from Canvas overlay to SVG ink.
   - Covers: `AC-1`
   - Verify: `pnpm --filter whiteboard-fe typecheck`

5. Run AC coverage and focused frontend verification.
   - Verify:
     `./.agents/skills/implement-feature-gsd/scripts/check-ac-coverage.sh specs/042-p3c-01-svg-ink-layer/acceptance.md frontend/src/canvas/shapes/__tests__/shapes.test.tsx frontend/src/canvas/shapes/__tests__/registry.test.ts frontend/src/canvas/layers/__tests__/SvgLayer.test.tsx`
   - Verify: `pnpm --filter whiteboard-fe test -- shapes.test.tsx registry.test.ts SvgLayer.test.tsx`
   - Verify: `pnpm --filter whiteboard-fe typecheck`

## Risks And Deviation Handling

- If tests reveal the existing shape registry cannot support point-heavy SVG elements without
  changing interaction semantics, stop and update this plan before coding further.
- If drawing-tool behavior is needed to satisfy a test, treat that as scope creep and defer to
  `P3C-02`/`P3C-03` rather than expanding this phase.
- Rollback is limited to removing the new ink shape util registrations and tests; no data contract
  migration is planned.
