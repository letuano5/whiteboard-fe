# Plan 05.10-01: Saved export from materialized server truth

Status: Completed 2026-07-02

## Scope

Route saved-document native export through backend server truth, share import/export normalization
and reporting helpers, and verify the export path is read-only.

## Implementation Slices

1. Add shared native-file normalization/reporting.
   - Keep strict top-level validation for kind/schema/room/camera/assets.
   - Filter unsupported malformed element objects with indexed skip reasons.
   - Preserve all valid element fields and asset references.
   - Verification: frontend/native helper tests for `AC-3` and `AC-4`.

2. Add backend saved native export endpoint.
   - Authenticate and resolve room access before loading the document.
   - Materialize from shared `SyncRoom`/repository path via `getOrCreateSyncRoom`.
   - Build a native document from server elements and room metadata.
   - Return `documentClock` for tests/clients and do no mutation.
   - Verification: backend export tests for `AC-1` and `AC-2`.

3. Wire frontend saved export to backend.
   - Local mode still exports from local store.
   - Saved mode calls the export API, downloads the returned document, and never uses stale local elements.
   - Verification: frontend API/component tests for `AC-1`.

4. Update backend import reporting.
   - Read payload through the shared normalizer.
   - Return imported/skipped counts and skip reasons.
   - Avoid executing replace for invalid payloads that cannot be safely normalized.
   - Verification: backend import tests for `AC-4`.

## Acceptance-Test Tasks

- `AC-1`: backend export endpoint and frontend saved export tests prove saved export uses server truth.
- `AC-2`: backend export test proves no documentClock mutation during export after a hot room commit.
- `AC-3`: shared native round-trip test preserves element metadata and asset references.
- `AC-4`: shared/backend import tests cover skipped unsupported objects and no half-mutated saved document.

## Risks And Deviation Handling

- Risk: frontend saved export previously had no backend export API. Mitigation: add a narrow endpoint
  under existing authenticated room file lifecycle routes and keep local export unchanged.
- Risk: lenient element skipping could mask invalid top-level files. Mitigation: skip only per-element
  object failures after the native file envelope, room, camera, and assets validate strictly.
