# Plan 02.51-01: Add Image / Background Map Support

## Objective

Implement `P2.5-01` so users can insert image elements from a URL or uploaded file, render them in
the existing SVG layer, and use current select/z-order behavior to treat them as background maps.

## Target Files

- `frontend/src/canvas/shapes/image.tsx`
- `frontend/src/canvas/shapes/index.ts`
- `frontend/src/components/toolbar/ImageInsertControl.tsx`
- `frontend/src/components/toolbar/image-insert.ts`
- `frontend/src/components/toolbar/Toolbar.tsx`
- `frontend/src/canvas/shapes/__tests__/registry.test.ts`
- `frontend/src/canvas/shapes/__tests__/shapes.test.tsx`
- `frontend/src/components/toolbar/__tests__/Toolbar.test.tsx`
- `frontend/src/canvas/tools/__tests__/select-tool.test.ts`
- `.planning/phases/02.51-p2-5-01-image-background/02.51-VERIFICATION.md`

## Tasks

1. Add and register `imageShapeUtil` backed by SVG `<image>` rendering, rectangular hit-test, and
   bounds support.
   - Covers: `AC-1`, `AC-2`, `AC-3`
   - Verify: `pnpm --filter whiteboard-fe test -- shapes.test.tsx registry.test.ts`

2. Add an image toolbar control that accepts URL input and uploaded image files, creates committed
   `image` elements through `createElement`, selects the created image, and places it below all
   visible elements by patching its `zIndex`.
   - Covers: `AC-1`, `AC-2`, `AC-4`
   - Verify: `pnpm --filter whiteboard-fe test -- Toolbar.test.tsx`

3. Add select-tool acceptance coverage proving images can be moved and resized through existing
   mutation-pipeline flows.
   - Covers: `AC-3`
   - Verify: `pnpm --filter whiteboard-fe test -- select-tool.test.ts`

4. Run AC coverage and focused frontend verification.
   - Verify:
     `./.agents/skills/implement-feature-gsd/scripts/check-ac-coverage.sh specs/046-image-background/acceptance.md frontend/src/canvas/shapes/__tests__/shapes.test.tsx frontend/src/canvas/shapes/__tests__/registry.test.ts frontend/src/components/toolbar/__tests__/Toolbar.test.tsx frontend/src/canvas/tools/__tests__/select-tool.test.ts`
   - Verify:
     `pnpm --filter whiteboard-fe test -- shapes.test.tsx registry.test.ts Toolbar.test.tsx select-tool.test.ts`
   - Verify: `pnpm --filter whiteboard-fe typecheck`

## Risks And Deviation Handling

- If URL/file insertion requires a broader asset-management model, stop and split that into a later
  asset phase rather than changing persistence contracts here.
- If toolbar space becomes too constrained, keep this phase to a compact icon button plus modal and
  defer larger tool-palette redesign.
- If select-tool behavior diverges for images, fix the shared bounds/hit-test path first; do not add
  a second image-specific move/resize implementation.
- Rollback is limited to removing the image ShapeUtil registration, toolbar control, tests, and GSD
  phase artifacts. No database migration is planned.
