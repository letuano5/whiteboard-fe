# Plan 05.9-01: Backend replace document command and import adapter path

Status: Completed 2026-07-02

## Scope

Implement P5-09 backend replace semantics: authoritative `ReplaceDocumentCommand`, fresh slot-clock
rebuild, tombstone old ids, room epoch bump, native import through sync room persistence, and
server-side `ROOM_REPLACED` broadcast payload.

## Implementation Slices

1. Add shared `ROOM_REPLACED` event and `RoomReplacedPayload`.
   - Verification: backend/shared typecheck.

2. Harden `ReplaceDocumentCommand` planning:
   - Normalize incoming elements to active state.
   - Increment `roomEpoch`.
   - Tombstone active ids absent from replacement.
   - Rebuild all incoming slot clocks from scratch.
   - Treat same-id different type/schema as a fresh record with no old slot-clock retention.
   - Verification: `sync-room-replace-document` tests.

3. Update `SyncRoom` persistence/apply path for replace:
   - Clear stale slot clocks for deleted ids and replacement ids before applying rebuilt clocks.
   - Persist tombstones and upsert incoming records through the P5 persistence transaction.
   - Produce replace payload data for broadcast.
   - Verification: `sync-room-replace-document` and `sync-room-persistence` tests.

4. Route native saved-document import through `ReplaceDocumentCommand` via the sync module, not
   `saveRoomElements`.
   - Permission check remains before command construction.
   - Verification: `native-file-import` tests prove no repository bypass.

5. Broadcast `ROOM_REPLACED` for non-replayed replace commits from the sync command handler/import
   bridge when socket context exists.
   - Verification: handler/import tests.

## Acceptance-Test Tasks

- `AC-1`: native import uses replace command/sync persistence instead of direct repository writes.
- `AC-2`: shared restore helper/path uses the same replace execution primitive and emits one replace payload.
- `AC-3`: stale `baseRoomEpoch` command after replace rejects `STALE_ROOM_EPOCH`.
- `AC-4`: unauthorized import is rejected before command construction/parsing application.
- `AC-5`: same-id type replacement rebuilds slot clocks from scratch.

## Risks And Deviation Handling

- Risk: snapshot restore storage is not present. Mitigation: implement reusable replace execution path
  and test a restore-flavored command; do not invent snapshot storage.
- Risk: HTTP import has no live socket to broadcast directly. Mitigation: expose replace payload/result
  from the sync module and keep socket handler broadcast behavior for realtime command path.
