## Implementation Slices

1. Add backend arrow binding helpers in `sync-room-planner.ts`:
   - Normalize object and legacy-string bindings to target references.
   - Compute endpoint points from target bounds and `anchorRatio`.
   - Build full `SlotPatch` entries for `binding.start`, `binding.end`, `geometry.startPoint`,
     `geometry.endPoint`, and `geometry.route` as needed.

2. Extend `planDeleteElements`:
   - Keep existing active/tombstone validation and `MAX_ELEMENTS_PER_DELETE`.
   - Find active arrows referencing deleted target ids.
   - Clear affected terminal bindings, preserve the endpoint as a concrete point, normalize geometry,
     and emit repair slot clocks at the command `serverClock`.
   - Enforce `MAX_REPAIRED_ARROWS_PER_COMMAND` and existing change-set byte limit.

3. Add `UpdateArrowBindingCommand` planning:
   - Validate arrow existence/type and binding target existence/type.
   - Preserve the opposite terminal from server-current arrow.
   - Recompute start/end points from current bindings and target state.
   - Return `reason: binding_update` or `repair` with full binding/geometry slot patches.
   - Treat stale geometry/binding clocks as a rebase-safe server recompute path.

4. Extend patch-slot planning for target movement/resize/rotation/linear geometry:
   - After applying command patches, detect target elements whose position/size/rotation/geometry
     changed.
   - Repair active arrows bound to those targets in the same plan and same `serverClock`.
   - Preserve unrelated concurrent patches and reject when repair count exceeds limit.

5. Add acceptance tests in `sync-room-delete-binding-repair.test.ts`, each tagged with `@covers AC-n`:
   - AC-1/AC-2 delete repairs bindings and emits slot patches.
   - AC-3 create cannot resurrect tombstoned id.
   - AC-4 repair/delete/change-set limits reject without partial commit.
   - AC-5 target transform repairs bound arrow in same clock.
   - AC-6 concurrent terminal binding updates preserve both terminals.
   - AC-7 missing/deleted binding target rejects.
   - AC-8 already tombstoned delete rejects and original retry replays ACK.

## Risks And Deviation Handling

- Legacy string bindings can appear in persisted data. If they cannot be safely recomputed, clear them
  on delete and avoid introducing a new migration path in this phase.
- If route/path recompute requires product-level routing rules, stop and ask; otherwise use straight
  start/end point geometry and `geometry.route: null`.
- If implementation requires changing shared contract shape, update `packages/shared` tests and
  rerun P5-02 contract checks before continuing.
