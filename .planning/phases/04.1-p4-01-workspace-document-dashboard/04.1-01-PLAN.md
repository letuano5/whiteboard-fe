# Plan 04.1-01: Implement document dashboard API, UI route, metadata schema, and AC tests

<objective>
Deliver P4-01 end to end: authenticated users can manage accessible saved documents from
`/dashboard`, anonymous users do not get document lists, and saved documents continue to open
through `/?room=<uuid>`.
</objective>

<tasks>

## T1 - Backend schema and room creation metadata

**Files:** `backend/prisma/schema.prisma`, new Prisma migration,
`backend/src/rooms/local-board-save.ts`, `backend/src/rooms/local-board-save.test.ts`

**Action:**

- Add Room metadata fields: `workspaceId`, `visibility`, `locked`, `archivedAt`,
  `lastOpenedAt`, `createdBy`.
- Add `RoomMember.lastOpenedAt`.
- Ensure local-board save creates `visibility: 'private'`, `createdBy`, and owner membership.

**Acceptance criteria:** AC-5

**Verify:** `pnpm --filter whiteboard-be test:run -- src/rooms/local-board-save.test.ts`

## T2 - Backend document dashboard API

**Files:** `backend/src/documents/document-dashboard.ts`,
`backend/src/documents/document-dashboard.test.ts`, `backend/src/app.ts`

**Action:**

- Add authenticated routes for listing dashboard documents, creating a document, opening a
  document, renaming, archiving, and deleting.
- List query must be access-scoped to `ownerId = userId` or `RoomMember(userId)`.
- Hide archived documents by default; include them only on archived filter.
- Reject management actions unless actor is owner/admin.

**Acceptance criteria:** AC-2, AC-3, AC-4, AC-5, AC-6, AC-7

**Verify:** `pnpm --filter whiteboard-be test:run -- src/documents/document-dashboard.test.ts`

## T3 - Frontend dashboard API client

**Files:** `frontend/src/documents/document-api.ts`,
`frontend/src/documents/__tests__/document-api.test.ts`

**Action:**

- Add typed client functions for listing documents, creating documents, opening documents,
  renaming, archiving, and deleting.
- Reuse `authenticatedFetch`.

**Acceptance criteria:** AC-5, AC-7

**Verify:** `pnpm --filter whiteboard-fe test -- src/documents/__tests__/document-api.test.ts`

## T4 - Frontend dashboard route and UI

**Files:** `frontend/src/app/App.tsx`, `frontend/src/app/bootstrap.ts`,
`frontend/src/app/__tests__/App.routing.test.tsx`,
`frontend/src/app/__tests__/bootstrap.test.ts`,
`frontend/src/documents/DocumentDashboard.tsx`,
`frontend/src/documents/__tests__/DocumentDashboard.test.tsx`

**Action:**

- Route `/dashboard` to the dashboard without initializing canvas socket/local-board sync.
- Show login/local-board actions for anonymous users instead of document groups.
- Show Owned, Shared with me, and Recent groups for authenticated users.
- Add search/status/archived filters and create/open/rename/archive/delete controls.

**Acceptance criteria:** AC-1, AC-3, AC-4, AC-5, AC-7

**Verify:** `pnpm --filter whiteboard-fe test -- src/app/__tests__/App.routing.test.tsx src/app/__tests__/bootstrap.test.ts src/documents/__tests__/DocumentDashboard.test.tsx`

</tasks>
