<objective>
Move saved-room frontend mutation sending and reconciliation onto the P5 command/pending-queue
path with durable drag flush, slot-aware replay, reconnect safety, and AC-tagged tests.
</objective>

<tasks>

1. Add P5 command queue primitives.
   - Files: `frontend/src/sync/socket/p5-command-queue.ts`, `frontend/src/sync/socket/state.ts`.
   - Action: define flush/backpressure constants, pending command state, mutation-to-command
     materialization, slot diff grouping, drag coalescing by `{ elementId, slot }`, dependency order
     for pending create/patch/delete, and single-slot undo metadata helpers.
   - Verify: `pnpm --filter whiteboard-fe test -- src/sync/socket/p5-command-queue.test.ts`.
   - Acceptance criteria: AC-1, AC-2, AC-6, AC-7.

2. Route saved-room mutation hooks through P5 commands.
   - Files: `frontend/src/sync/socket/subscriptions.ts`.
   - Action: replace outbound saved-room `ELEMENT_UPDATE` mutation writes with enqueue/flush of
     `WS_EVENTS.SYNC_COMMAND`; keep cursor/selection/draft preview events ephemeral.
   - Verify: command queue tests assert no presence/draft action creates SyncCommand.
   - Acceptance criteria: AC-1, AC-8.

3. Connect ACK/diff/reconnect lifecycle to the pending command queue.
   - Files: `frontend/src/sync/socket/event-handlers.ts`,
     `frontend/src/sync/socket/p5-reconciliation.ts`,
     `frontend/src/sync/socket/state.ts`.
   - Action: clear processed pending commands, avoid double replay after reconnect, ignore stale
     ACK clocks except for matching pending cleanup, and replay pending commands only in dependency
     order after server state is applied.
   - Verify: `pnpm --filter whiteboard-fe test -- src/sync/socket/p5-reconciliation.test.ts`.
   - Acceptance criteria: AC-3, AC-4, AC-5, AC-7.

4. Run AC coverage and package checks.
   - Files: `specs/040-p5-11-frontend-reconciliation/acceptance.md`.
   - Action: ensure every `AC-n` has at least one `@covers AC-n` test tag, run coverage guard, then
     run focused frontend typecheck/tests.
   - Verify:
     `.agents/skills/implement-feature-gsd/scripts/check-ac-coverage.sh specs/040-p5-11-frontend-reconciliation/acceptance.md frontend/src/sync/socket/p5-command-queue.test.ts frontend/src/sync/socket/p5-reconciliation.test.ts`
     and `pnpm --filter whiteboard-fe typecheck`.
   - Acceptance criteria: AC-1 through AC-8.

</tasks>

<risks>

- Existing legacy tests may still assert `ELEMENT_UPDATE` saved-room writes. If they conflict with
  P5-11, update those tests to assert the new `SYNC_COMMAND` path while preserving local/cross-tab
  legacy behavior.
- Backend validation rejects `order` slot patches, so frontend diffing must not emit direct order
  patches from normal element patch events.
- Full optimistic/server dual-store UI state is broad; this plan scopes it to explicit pending
  command replay over the existing displayed element store and clock metadata.
  </risks>

<success_criteria>

- Saved-room mutations are queued as P5 commands with bounded durable drag behavior.
- Slot-aware reconciliation preserves independent peer changes and ignores stale ACK payload state.
- Reconnect status handling never double-applies processed commands and preserves pending
  dependency order.
- All AC-1 through AC-8 are covered by tagged tests and the AC coverage guard passes.
  </success_criteria>
