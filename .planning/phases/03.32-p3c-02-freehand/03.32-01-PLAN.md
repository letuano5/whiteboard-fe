# Plan 03.32-01: Implement Freehand Drawing

## Objective

Implement `P3C-02` so users can draw SVG freehand strokes that commit through the existing mutation
pipeline, use simplified point geometry, and split automatically at the per-shape point cap.

## Risks And Deviation Handling

- If existing pointer handling bypasses the mutation pipeline for creates, stop and update this
  plan before implementing a parallel write path.
- If SVG path rendering cannot use simplified points without breaking line/arrow point rendering,
  isolate simplification to ink elements and update tests before continuing.
- If point-cap auto-commit conflicts with undo/redo semantics, keep each committed stroke as a
  normal element creation and document any follow-up undo grouping work instead of introducing a
  new history model in this phase.
- Rollback is limited to removing freehand tool routing, helper functions, and tests; no data
  migration or backend schema changes are planned.
