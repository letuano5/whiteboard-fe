## Implementation Slices

1. Add shared sync slot vocabulary and field mapping.
   - Define `SyncSlot`, semantic slot value types, `ELEMENT_FIELD_SYNC_CLASSIFICATION`, and
     `ELEMENT_PROPS_FIELD_SYNC_CLASSIFICATION`.
   - Export from `packages/shared/src/index.ts`.
   - Verification: `pnpm --filter @vdt/shared typecheck`.

2. Add shared command and change-set contracts.
   - Define `SlotPatch`, `CreateElementCommand`, `PatchSlotsCommand`,
     `ReorderElementsCommand`, `UpdateArrowBindingCommand`, `DeleteElementsCommand`,
     `ReplaceDocumentCommand`, `SyncCommand`, read preconditions, order hints, and
     `CommittedChangeSet`.
   - Ensure command envelopes include protocol/schema/room/request/client/epoch metadata and
     exclude actor identity.
   - Verification: `pnpm --filter @vdt/shared typecheck`.

3. Add shared validation helpers.
   - Validate patch slot values, reject duplicate patch slots, reject `order` patching,
     reject `isDeleted` patching, validate command envelopes, validate create duplicate/tombstone
     preconditions, and materialize a create result for later server clock initialization.
   - Verification: `pnpm --filter whiteboard-be test -- shared-contracts`.

4. Add AC-tagged tests.
   - Test every `AC-n` from `acceptance.md`.
   - Run AC coverage script and relevant typecheck/tests.

5. Keep shared package file-size guardrails.
   - Move the existing native-file validation block out of `index.ts` into `native-file.ts`
     while preserving public exports.
   - Verification: `pnpm --filter @vdt/shared typecheck`.

6. Reconcile the revised `docs/SPECS.md` P5-02 contract details.
   - Add `ArrowEndpointBinding`, `SlotReadPrecondition.onStale`, slot clock stale validation,
     command-level-only request/ACK constraints, the revised `UpdateArrowBindingCommand` shape, and
     `manual_replace` replace reasons.
   - Verification: `pnpm --filter whiteboard-be test -- shared-contracts`.
