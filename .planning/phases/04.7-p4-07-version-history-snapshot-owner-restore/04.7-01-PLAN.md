# Plan 04.7-01: Version history snapshots and owner restore

Status: Completed 2026-07-03

## Objective

Implement P4-07 saved-room version history with automatic interval snapshots, import/restore
safety snapshots, owner-only restore through `SyncRoom`, and a compact saved-room history panel.

## Tasks

1. Add snapshot persistence schema and migration.
   - Files: `backend/prisma/schema.prisma`,
     `backend/prisma/migrations/20260703100000_add_snapshots/migration.sql`
   - Acceptance: AC-1, AC-5, AC-6
   - Verify: `pnpm --filter whiteboard-be typecheck`

2. Implement backend snapshot service and HTTP router.
   - Files: `backend/src/rooms/room-history.ts`, `backend/src/app.ts`
   - Behavior: list metadata, capture snapshots from materialized truth, auto interval capture,
     owner-only restore via `executeReplaceDocument`, and no manual snapshot route.
   - Acceptance: AC-1, AC-2, AC-3, AC-5, AC-6, AC-7, AC-8
   - Verify: `pnpm --filter whiteboard-be test:run src/rooms/room-history.test.ts`

3. Wire safety snapshots and interval trigger.
   - Files: `backend/src/rooms/native-file-import.ts`, `backend/src/sync/sync-room-persistence.ts`
   - Behavior: import captures `import_safety` before replace; persistence attempts interval
     snapshots after successful saved-room commits.
   - Acceptance: AC-5, AC-6
   - Verify:
     `pnpm --filter whiteboard-be test:run src/rooms/native-file-import.test.ts src/rooms/room-history.test.ts`

   - Acceptance: AC-1, AC-2, AC-7

## Risks And Deviation Handling

- Risk: Prisma generated types may not expose `snapshot` until client generation. Mitigation:
  type the minimal delegate contract in `room-history.ts` and verify with TypeScript.
- Risk: automatic snapshot attempts must not make document commits fail. Mitigation: interval
  snapshot hook catches/logs errors after the commit succeeds.
- Risk: restore must not bypass authoritative sync. Mitigation: tests mock `executeReplaceDocument`
  and assert no `record`/`tombstone` delegate mutation is called by restore service.
