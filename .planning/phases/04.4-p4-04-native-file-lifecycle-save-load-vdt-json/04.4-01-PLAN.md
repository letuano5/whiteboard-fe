<objective>
Implement P4-04 native `.vdt.json` save/load lifecycle. Users can export current local or saved
board data, import a validated native file into local state or a permitted saved document, see clear
validation/permission errors, and confirm before replacing existing data.
</objective>

<context>
- Canonical scope: `docs/SPECS.md` `[P4-04] Native file lifecycle: save/load .vdt.json`.
- Acceptance registry: `specs/029-native-file-lifecycle/acceptance.md`.
- Prior phases established root-path local board mode, saved documents at `/?room=<uuid>`, local
  persistence, authenticated local-to-saved conversion, and server-side `effectiveRole` checks.
- Persisted import must use the existing room persistence path, not direct table writes.
</context>

<success_criteria>

- `.vdt.json` export/import round-trips existing board data and metadata.
- Anonymous import remains local-only and does not create a DB room.
- Viewer saved-room import is rejected; editor/owner import is allowed.
- Invalid native files show clear validation errors and do not crash.
- Existing board data is not replaced or merged without explicit confirmation.

</success_criteria>

<risks>
- Import replace semantics must not accidentally resurrect deleted/tombstoned elements outside the
  repository path.
- FileReader and Blob download behavior need focused unit tests because jsdom lacks full browser
  download fidelity.
- Backend schema validation must avoid accepting arbitrary JSON that later fails deep inside Prisma.
</risks>

<deviation_handling>
If implementation reveals a missing product decision around merge semantics, creating a new saved
document directly from import, or asset metadata restoration, stop source edits, update
`CONTEXT.md` and this plan, then ask the user in Vietnamese before continuing.
</deviation_handling>
