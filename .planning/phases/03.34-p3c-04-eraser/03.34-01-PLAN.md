# Plan 03.34-01: Implement Eraser Tool

## Objective

Implement `P3C-04` so users can drag an eraser through existing SVG-rendered elements and delete
whole shapes through the shared mutation pipeline while reusing registered shape hit-test utilities.

## Target Files

- `frontend/src/canvas/tools/eraser-tool.ts`
- `frontend/src/canvas/tools/__tests__/eraser-tool.test.ts`
- `frontend/src/canvas/hooks/use-whiteboard-pointer-handlers.ts`
- `frontend/src/components/toolbar/Toolbar.tsx`
- `frontend/src/components/toolbar/__tests__/Toolbar.test.tsx`
- `specs/044-p3c-04-eraser/acceptance.md`
- `.planning/phases/03.34-p3c-04-eraser/VERIFICATION.md`

## Tasks

1. Add eraser hit-test helpers that sweep between consecutive pointer samples and call registered
   shape utils for candidate elements.
   - Covers: `AC-3`
   - Verify: `pnpm --filter whiteboard-fe test -- eraser-tool.test.ts`

2. Route eraser pointer down/move/up/leave through the existing SVG pointer handlers and delete
   matching visible elements using `deleteElements`.
   - Covers: `AC-1`, `AC-2`, `AC-4`
   - Verify: `pnpm --filter whiteboard-fe test -- eraser-tool.test.ts`

3. Add toolbar access for the eraser tool without changing view-only edit gating.
   - Covers: `AC-1`
   - Verify: `pnpm --filter whiteboard-fe test -- Toolbar.test.tsx`

4. Run AC coverage and focused frontend verification.
   - Verify:
     `./.agents/skills/implement-feature-gsd/scripts/check-ac-coverage.sh specs/044-p3c-04-eraser/acceptance.md frontend/src/canvas/tools/__tests__/eraser-tool.test.ts frontend/src/components/toolbar/__tests__/Toolbar.test.tsx`
   - Verify:
     `pnpm --filter whiteboard-fe test -- eraser-tool.test.ts Toolbar.test.tsx`
   - Verify: `pnpm --filter whiteboard-fe typecheck`

## Risks And Deviation Handling

- If registered shape utils do not expose enough geometry for segment sweeping, approximate the
  sweep with sampled points that call the existing `hitTest` function and document the tradeoff in
  verification.
- If realtime sync hooks do not observe `deleteElements`, stop and update the plan before adding a
  parallel delete path.
- Rollback is limited to removing the eraser tool routing, toolbar entry, tests, and this phase's
  artifacts.
