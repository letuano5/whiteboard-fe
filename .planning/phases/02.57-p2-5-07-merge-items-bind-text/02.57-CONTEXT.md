# Phase 2.57: P2.5-07 Merge items + bind text vào container - Context

**Gathered:** 2026-07-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Provide a "merge items" MVP: multi-select 2+ elements into a persistent unit of operation via a
shared `groupId` (Merge/Unmerge commands), with cascade behavior for move/resize/delete/copy/
z-order applied to the whole group. The most important special case is binding exactly one `text`
element to exactly one container-eligible shape (rectangle/ellipse/diamond/triangle/polygon/image)
so the text acts as a centered, width-wrapped label that follows the container's move/resize/
delete/z-order. No boolean-union/flatten-shape geometry, no nested groups, no rotate-by-bbox.

</domain>

<decisions>
## Implementation Decisions

### Group selection UX
- **D-01:** Clicking one element that belongs to a group selects the **whole group** as one unit
  — combined bbox + move/resize handles appear, matching Figma-style grouped selection. This is
  the trigger mechanism for the AC-9 group-bbox resize operation. A separate interaction (e.g.
  double-click) is needed to drill into and select/edit a single element inside the group; the
  planner should define the exact drill-in gesture during planning/implementation (not re-asked
  here since it's a standard, low-ambiguity affordance).

### Locked elements + merge
- **D-02:** A `locked` element cannot be added to a Merge selection — Merge is a no-op / excluded
  for locked elements. If an element inside an existing group becomes locked afterward, group
  cascade (move/delete) must skip that locked member — it stays in place and is not deleted when
  the rest of the group moves/deletes.

### Independent text font scaling on group resize
- **D-03:** Text elements that are part of a group but **not** bound to a container (i.e. no
  container-eligible sibling to bind to) do **not** scale `fontSize` when the group is resized by
  its combined bbox. Only their `(x, y)` position is translated/scaled to the new bbox; `fontSize`
  stays fixed. This applies only to independent group text — container-bound text still
  recenters/rewraps per AC-6/AC-9 (that binding behavior is unaffected by this decision).

### Plan slicing (delivery cadence)
- **D-04:** `gsd-plan-phase` must structure this phase as **3 sequential plan slices**, matching
  the sub-phase breakdown already recommended in `docs/SPECS.md` `[P2.5-07]` "Ghi chú triển khai":
  1. **Slice (a):** Merge/Unmerge commands (context menu + Cmd/Ctrl+G, Cmd/Ctrl+Shift+G), join-
     existing-group vs flatten-multi-group rule, group-aware delete cascade, and the copy/
     duplicate `groupId` remap fix (duplicate currently carries `groupId` through unchanged —
     `frontend/src/canvas/tools/select/clipboard.ts` `cloneAsNewDraft` L17 — must instead assign a
     fresh shared `groupId` to duplicated group members).
  2. **Slice (b):** Text-binding at merge time — center/wrap text into exactly one container,
     `zIndex` above the container, z-order command propagation to bound text, move/resize/delete
     following behavior for the bound pair.
  3. **Slice (c):** Group-bbox resize for multi-element groups — non-text scaling, container-bound
     text recenter/rewrap, independent text position-only scaling per D-03.
  Each slice should be its own plan/commit so it can be reviewed and rolled back independently.

### Claude's Discretion
- Exact drill-in gesture to select a single element inside an already-selected group (e.g.
  double-click, or a modifier+click) — planner decides based on existing interaction patterns in
  `frontend/src/canvas/tools/select/`.
- Text wrapping/measurement implementation approach (no existing utility in the codebase — must be
  built from scratch; e.g. canvas 2D `measureText` line-breaking + SVG `<tspan>` rendering vs SVG
  `foreignObject` with a wrapping div). This is a technical implementation choice for research/
  planning, not a product decision.
- Exact keyboard-shortcut matcher wiring for Cmd/Ctrl+G / Cmd/Ctrl+Shift+G, following the existing
  pattern in `frontend/src/canvas/shortcuts/select-shortcuts.ts` and `shortcut-matchers.ts`.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `getMultiSelectBounds(elements, selectedIds)` — `frontend/src/canvas/layers/svg/selectors.ts`
  L89-109. Computes combined min/max bbox over `x/y/width/height`; reusable directly for group
  bbox resize (AC-9) by passing group member ids instead of `selectedIds`.
- `computeMultiDragDrafts` — `frontend/src/canvas/tools/select/bound-arrows.ts` L43-70. Existing
  multi-element drag-by-delta implementation (also drags bound arrows); the pattern to extend for
  "drag any group member moves the whole group" (AC-8).
- `computeBoundArrowDrafts` — `frontend/src/canvas/tools/select/bound-arrows.ts` L14-38. Existing
  "look up all elements referencing X, patch them" pattern — the closest analog for propagating
  container z-order/move/resize to bound text, and for remapping `groupId` on duplicate.
- `frontend/src/store/zorder.ts` L5-62 — `bringToFront`/`sendToBack`/`bringForward`/
  `sendBackward`. Currently single-element only; must be extended so a z-order command on a bound
  container also carries its bound text (AC-7), and is a no-op when invoked directly on bound text.
- `generateId()` — `frontend/src/utils/id.ts` L1. Use for new `groupId` values.

### Established Patterns
- All committed mutations flow through `frontend/src/store/mutation-pipeline.ts`
  (`createElements`/`patchElement`/`updateElements`/`deleteElements`) — group mutations must batch
  through this pipeline as one commit (AC-8, AC-12 undo/redo as one step), not per-element calls.
- Context menu commands live in `frontend/src/components/context-menu/ContextMenu.tsx` (currently
  z-order only, L82-109); the multi-select gate to remove is `const disabled = selectedCount !== 1
  || !selectedId` at L33 (per `docs/SPECS.md` explicit instruction to lift this gate for Merge).
- Keyboard shortcuts for the select tool live in
  `frontend/src/canvas/shortcuts/select-shortcuts.ts` (copy/paste/duplicate/delete pattern to
  follow for Cmd/Ctrl+G and Cmd/Ctrl+Shift+G).

### Integration Points
- `frontend/src/canvas/tools/select/clipboard.ts` `cloneAsNewDraft` L6-22 — duplicate currently
  passes `groupId` through unchanged (L17); this is the exact spot to fix for AC-11 (fresh
  `groupId` on copy, no reuse of the original).
- `frontend/src/canvas/tools/select/pointer-move.ts` L37-43 — branches on `selectedIds.length > 1`
  for existing multi-drag; group-aware drag (AC-8) extends this branch to resolve full group
  membership from `groupId`, not just the current `selectedIds` set.
- No text wrapping/measurement utility exists (`frontend/src/canvas/tools/text-editor.tsx` uses
  `contentEditable` with `whiteSpace: 'pre'`, no wrap) — AC-4's wrap-to-container-width behavior is
  new code, not an extension of existing text editing.
