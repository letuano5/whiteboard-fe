## Group selection UX

| Option | Description | Selected |
|--------|-------------|----------|
| Chọn cả group | Click 1 element trong group → toàn bộ group được select như 1 khối (bbox tổng hợp + handle resize/move hiện ra), giống Figma. Double-click (hoặc thao tác riêng) để chọn/sửa 1 element lẻ bên trong group. | ✓ |
| Chỉ chọn element đó | Click chỉ chọn đúng element đó; move/delete vẫn cascade cả group ở background, nhưng resize theo bbox group cần một cách trigger riêng. | |

**User's choice:** Chọn cả group (khuyến nghị).
**Notes:** This is the trigger mechanism needed for AC-9 group-bbox resize; drill-in gesture for
editing a single element inside the group left to Claude's discretion during planning.

## Independent text font scaling on group resize

| Option | Description | Selected |
|--------|-------------|----------|
| Không scale font | Giữ nguyên fontSize khi group resize; chỉ vị trí (x,y) của text được scale theo bbox mới. | ✓ |
| Scale tuyến tính theo group | fontSize nhân theo tỉ lệ resize trung bình; chấp nhận rủi ro tràn/chữ quá nhỏ ở trường hợp resize lệch tỉ lệ mạnh. | |

**User's choice:** Không scale font (khuyến nghị).
**Notes:** `docs/SPECS.md [P2.5-07]` phrased this vaguely ("có thể scale tuyến tính nếu không làm
chữ tràn xấu") — this decision locks a concrete, simpler rule. Container-bound text recenter/
rewrap behavior (AC-6/AC-9) is unaffected — it already has explicit rules.
