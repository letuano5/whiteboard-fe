### `frontend/src/canvas/tools/select/group-resize.ts` (utility/transform, CRUD batch)

**Analog:** `getMultiSelectBounds` in `frontend/src/canvas/layers/svg/selectors.ts` (lines 89-109) for bbox math; `frontend/src/canvas/tools/select/resize.ts` for the "why not reuse resize session" boundary (see Pitfall 4 in RESEARCH.md — do NOT model this on `resizeBoundsFromAnchorAndPointer`, it's anchor/handle driven, structurally different).

**Bbox computation to reuse directly** (selectors.ts lines 89-109):
```typescript
export function getMultiSelectBounds(
  elements: Element[],
  selectedIds: string[],
): MultiSelectBounds | null {
  if (selectedIds.length <= 1) return null;
  const selectedElements = elements.filter((el) => selectedIds.includes(el.id) && !el.isDeleted);
  if (selectedElements.length <= 1) return null;
  const xs = selectedElements.flatMap((el) => [el.x, el.x + el.width]);
  const ys = selectedElements.flatMap((el) => [el.y, el.y + el.height]);
  const minX = Math.min(...xs);
  const minY = Math.min(...ys);
  return { x: minX, y: minY, width: Math.max(...xs) - minX, height: Math.max(...ys) - minY };
}
```
Call this with group member ids (from `resolveGroupMembers`) instead of `selectedIds` to get the old bbox; new group-resize math is a from-old-bbox-to-new-bbox linear transform of each member's `x/y/width/height` (new code, no existing analog for the transform itself — write fresh per RESEARCH.md's explicit note that this is a different computation shape than single-element resize).

**Non-text vs text branching per D-03:** independent group text (no binding) gets position-only translate/scale (no `fontSize` change); bound text (via `resolveGroupBinding`) recenters/rewraps using `text-wrap.ts`. Both branches must still be one flat array of patches into a single `updateElements` call.

export function bringToFront(id: string): void {
  const { elements } = useElementsStore.getState();
  const active = elements.filter((e) => !e.isDeleted);
  const target = active.find((e) => e.id === id);

}
