# Plan 03.33-01: Implement Highlighter Drawing

## Objective

Implement `P3C-03` so users can draw SVG highlighter strokes that commit through the existing
mutation pipeline, use fixed semi-transparent/wider defaults, and reuse the freehand point pipeline.

## Risks And Deviation Handling

- If generalizing freehand drawing would destabilize current freehand behavior, keep the public
  freehand functions as wrappers and add highlighter wrappers over the shared internal ink helpers.
- If existing toolbar layout becomes cramped, add the highlighter button within the current icon
  toolbar and leave broader responsive toolbar redesign for a later UI phase.
- If point-cap splitting creates awkward undo grouping, keep each committed highlighter stroke as a
  normal element creation matching freehand semantics rather than introducing a new history model.
- Rollback is limited to removing highlighter tool routing, toolbar entry, tests, and this phase's
  planning artifacts; no data migration or backend schema changes are planned.
