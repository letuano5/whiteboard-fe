# Plan 05.5-01: Implement change sets, ACK/reject/rebase, and broadcast primitives

## Scope

Implement `P5-05` from `docs/SPECS.md`: shared change-set/ack/broadcast contracts, backend
`SyncRoom` committed change-set metadata, ACK creation for commit/rebase/reject, and client-side
reconciliation helpers that clear pending, apply slot patches, avoid whole-element replacement for
slot-only changes, ignore stale clocks, and request diffs on gaps.

## Implementation Slices

1. Extend shared sync contracts with `ChangeSetReason`, canonical `CommittedChangeSet`
   fields, `SyncAck`, `SyncBroadcast`, and stale/gap result types.
   - Verification: `pnpm --filter @vdt/shared test -- sync`.

2. Update backend `SyncRoom` change-set generation with origin actor/request metadata,
   reason inference, `slotPatches`, `puts`, and `deletes`, preserving existing aliases needed by
   P5-03/P5-04 tests.
   - Verification: `pnpm --filter whiteboard-be test -- sync-room`.

3. Add ACK helper that returns `commit`, `rebase`, or `reject` payloads from execution results and
   command errors.
   - Verification: AC-tagged backend ACK tests.

4. Add frontend P5 reconciliation helper for ACK/broadcast processing and pending request state.
   - Verification: AC-tagged frontend tests for pending clear, reject behavior, rebase no-repush,
     same-origin broadcast, slot-only application, stale clock ignore, and gap diff request.

5. Run AC coverage and relevant package checks.
   - Verification: AC coverage script, focused tests, typecheck.

## Acceptance-Test Tasks

- `AC-1`: test commit/rebase ACK clears only the matching pending request.
- `AC-2`: test same-origin broadcast clears pending when ACK is missing.
- `AC-3`: test reject clears the matching request while retaining newer pending entries.
- `AC-4`: test rebase applies the server change-set without emitting/resending the command.
- `AC-5`: test slot-only change-set updates only the touched slot and preserves untouched element
  object fields.

## Risks And Deviation Handling

- Risk: existing P5-03/P5-04 tests depend on legacy `created/patched/deleted` change-set fields.
  Mitigation: add canonical P5 fields while keeping compatibility aliases until later cleanup.
- Risk: frontend still sends legacy element updates. Mitigation: keep P5 reconciliation helper
  isolated and covered; full frontend command migration remains P5-11.
- Risk: durable replay semantics belong to P5-06. Mitigation: ACK helpers expose wire payloads but
  do not claim persisted idempotency.
