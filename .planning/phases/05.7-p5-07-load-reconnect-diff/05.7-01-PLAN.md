# Plan 05.7-01: Implement P5 Snapshot/Diff Reconnect

## Objective

Implement P5-07 load/reconnect/diff behavior end-to-end across shared contracts, backend
repository/socket handlers, and frontend reconciliation helpers.

## Acceptance Mapping

- AC-1: Backend snapshot payload tests and frontend snapshot handler tests.
- AC-2: Backend diff handler/repository tests.
- AC-3: Backend reconnect request/status tests and frontend diff request emission tests.
- AC-4: Backend pending status classifier tests.
- AC-5: Repository diff boundary tests for `roomEpoch` and stale history wipe-all.
- AC-6: Repository slot clock filter tests.
- AC-7: Frontend reconciliation tests for deferred `lastServerClock` and known slot clock storage.
- AC-8: Frontend slot-aware `ROOM_DIFF` apply tests.

## Implementation Slices

1. Shared contracts
   - Add `RoomSnapshot`, `RoomDiff`, `ReconnectRequest`, `ReconnectResponse`,
     `PendingRequestStatus`, and related aliases to `packages/shared/src/sync-contracts/types.ts`.
   - Include protocol/schema versions, `serverClock`, `roomEpoch`, slot clocks, and wipe-all fields.
   - Verification: shared typecheck and focused shared contract tests.

2. Backend repository and status classification
   - Extend `RoomDiffResult` and `LoadRoomResult` to expose `serverClock`, `roomEpoch`,
     `fromClock`, `toClock`, `hasMore`, and optional history metadata.
   - Update `getRoomDiff` to read `targetClock`, reject replace-boundary diffs with wipe-all,
     and filter `slotClocks` per AC-6.
   - Add pending status classification from `ProcessedRequest`; unknown IDs return `unknown`.
   - Verification: `backend/src/persistence/room-repository.test.ts` plus focused status tests.

3. Backend realtime handlers
   - Update join/load snapshot payload shape.
   - Update `ROOM_DIFF_REQUEST` handler to accept `lastServerClock`, `roomEpoch`, and
     `pendingRequestIds`, emit `ROOM_DIFF` or wipe-all `ROOM_SNAPSHOT`, and include
     `pendingRequests`.
   - Keep old `fromClock` accepted as a compatibility alias only if needed by old callers.
   - Verification: `backend/src/realtime/handlers/room-diff-request.test.ts`,
     `backend/src/persistence/socket-reconnect.test.ts`.

4. Frontend snapshot/diff apply
   - Upgrade socket payload types and handlers to P5 payloads.
   - Add helper(s) to materialize server state, known slot clocks, and slot-aware diff patches.
   - Update `lastServerClock` only after full snapshot/diff application.
   - Ensure diff apply does not require `originRequestIds`.
   - Verification: `frontend/src/sync/socket/p5-reconciliation.test.ts` and socket handler tests.

5. Final verification and GSD closeout
   - Add/adjust tests tagged `@covers AC-1` through `@covers AC-8`.
   - Run `check-ac-coverage.sh`.
   - Run focused package tests, `pnpm typecheck`, and update `.planning` verification/summary.

## Risks And Deviation Handling

- Legacy P3A reconnect tests may assert `documentClock`; preserve a compatibility alias or update
  only the P5-specific tests if old assertions conflict with the SRS.
- If existing frontend store cannot yet maintain separate P5 `serverState`, implement the smallest
  local server clock/slot-clock tracking surface needed for P5-07 and leave full P5-11 migration out
  of scope.
- If chunking becomes necessary for `MAX_DIFF_BYTES`, keep the contract fields and return
  `hasMore: false`; do not implement stream/chunk pagination unless current tests or code require it.
